你所不知道的javascript之window对象上的一些常用方法

  • javascript

除了前面提到过的引用类型和基本包装类型以外，在javascript中还存在着一个顶级的全局对象，在浏览器端它被实现为window这个全局对象，很多我们平时直接使用的属性方法都是挂载在window这个对象上的，比如前面提到过的parseInt()，还有我们再熟悉不过的undefined，其实是window对象上的一个属性！另外还有alert()，setTimeout()等等。所以，今天我们来聊一聊那些挂载在window上的常用的辅助函数都有哪些。

javascript中的编码问题

  • escape()方法
    用法： escape(str)
    该方法返回一个可在所有计算机上读取的编码 String 对象。
    它可以使用类似 "%xx" 编码形式（其中 xx 等于表示该字符的十六进制数，字符值大于 255 的字符以 %uxxxx 格式存储）替换一些字符，而不会被此方法编码的字符有 "@" "*" "/" "+" "-" "_" "."

对应的解码方法：unescape(str)：从用 escape 方法编码的 String 对象中返回已解码的字符串。

  • encodeURI()方法
    用法： encodeURI(str)
    该方法返回编码为有效的统一资源标识符 (URI) 的字符串。
    encodeURI 方法一般用于对一个 url 进行整体上的编码。不会被此方法编码的字符有 "!" "@" "#" "$" "&" "*" "(" ")" "=" ":" "/" ";" "?" "+" "'"
    如果将编码结果传递给 decodeURI(str)，则将返回初始的字符串。

  • encodeURIComponent()方法
    用法： encodeURIComponent(str)
    该方法返回编码为统一资源标识符 (URI) 的有效组件的字符串。
    encodeURIComponent 方法一般用于对 url 中的请求参数名称和参数值进行编码，相当于对请求参数包含的特殊字符进行转义。不会被此方法编码的字符有 "!" "*" "(" ")" "'" "."
    如果将编码结果传递给 decodeURIComponent(str)，则将返回初始的字符串。

JSON

首先需要明确的一点是，json有自己的数据类型：字符串（字符串必须使用双引号）、数值、布尔值和null（不支持javascript中的undefined值），另外还支持对象（对象属性名称必须加双引号）和数组，不支持函数。
javascript中为json格式数据的解析和序列化提供了JSON对象，该对象上主要有两个方法：

  • JSON.parse()
    用法：JSON.parse(string[, function(key, value) {}])
    调用JSON.parse(str)解析字符串时，单引号内部必须是一个有效的json格式的数据，即必须严格按照上面的规则来定义，否则会报错！

可以向JSON.parse函数中传入函数作为第二个参数，传入的函数在解析得到的对象键值对上调用，该函数必须返回一个值，如果返回的是undefined，则会从结果中删除对应的键；否则会将返回的结果插入到最后解析得到的javascript对象中。

用法：JSON.stringify(obj[, filter, indentOrNot])

定时器

我们都知道，javascript中提供了两个函数来实现类似定时器的功能：

  • setTimeout(funciton(){}, timeout)：超时调用函数，返回一个定时器的ID值，可用于取消该定时器。
  • setInterval(function(){}, interval)：重复间隔调用函数，同样返回一个ID值。
    清除定时器：clearTimeout(timer)或者clearInterval(timer):只能取消在队列中尚未执行的定时器，如果要取消的定时器正在执行，则无法取消。

关于setTimeout()和setInterval()的区别：

  • 很明显，两者的功能不同，一个超时调用，一个则是重复间隔调用
  • 使用setTimeout()嵌套可以实现类似setInterval()的功能，但是两者还是有区别的：
    setInterval(function, interval)：本意上是每隔interval毫秒的时间间隔执行一次回调函数，但这是回调函数执行时间小于设置的interval毫秒的时间间隔的情况。如果回调函数的执行时间超过了interval毫秒的时间间隔，则在回调函数的执行过程中按interval毫秒的时间间隔将下一个回调函数添加到执行队列中，如果当前执行队列中存在一个尚未执行的定时器代码实例，则不会添加回调函数添加到执行队列中，这样可能会丢失一些回调函数。而setTimeout函数嵌套则不会发生这样的情况：

在setTimeout函数嵌套中，无论中间的代码执行经过多长时间，都会在每次执行操作后经过timeout毫秒的时间间隔之后将回调函数添加到执行队列中，然后执行，确保不会错过任何的回调函数。

其他函数

  • confirm()
    confirm() 通常用于弹出对话框(包括一个确定按钮和一个取消按钮)。
    用法：confirm(str)
    str参数表示要在消息对话框中显示的文本。函数返回一个布尔值，当用户点击”确定”按钮时，返回true；当用户点击”取消”按钮时，返回false。
  • prompt()
    prompt() 通常用于询问一些需要与用户交互的信息。弹出消息对话框（包含一个确定按钮、取消按钮与一个文本输入框）。
    用法：prompt(str1, str2)
    str1参数表示要显示在消息对话框中的文本，不可修改；str2参数表示需要用户输入的文本框中显示的内容，可以修改。函数的返回值：当点击确定按钮，文本框中的内容将作为函数返回值；点击取消按钮，将返回null。
