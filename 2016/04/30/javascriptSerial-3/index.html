你所不知道的javascript之包装类型

  • javascript

趁热打铁，让我们接着向javascript中另外的几种对象类型做进一步的认识。它们就是javascript中的基本包装类型，说是包装类型是有原因的，因为它们来源于javascript中的基本类型，只是做了适当的包装就变成了对象类型。下面让我们欢迎今天的嘉宾隆重登场，他们就是Number，Boolean和String。

  • Number类型
  • Boolean类型
  • String类型

Number类型

Number对象自身具有一些方法，最常用的就是 toFixed(num)这个方法了。它主要用于保留数值指定位数的小数，对于多余的小数部分则进行四舍五入的方法进行取舍。对于Number，String这些对象上的方法，作为对应的基本类型number和string是可以直接使用的，这其中就涉及到基本包装类型对象的自动创建问题，而在执行完基本类型调用引用类型方法的这一行代码时，自动创建的基本包装类型对象又会被自动销毁。

String类型

在实际的使用中，我们经常要对字符串进行处理，所以String类型提供了大量有用的方法供我们使用。但在实际过程中我们并不需要通过创建String对象来使用这些方法，而是直接在基本类型string上使用这些方法：

  • charAt(index)：返回指定索引处的字符
  • charCodeAt(index)：返回指定索引处的字符对应的字符编码
  • concat(str)：连接字符串，返回新的字符串，而原字符串保持不变
  • 截取字符串的方法（返回新的字符串，而原字符串保持不变）：
  • slice(start, [end])：截取从start到end - 1处的字符串。其中start参数必须小于end参数。当传入的参数为负数时，会与字符串的长度相加转化为非负数。

  • substring(start, [end])：作用跟slice一样。只是在传入的参数为负数的时候有区别。substring方法会统一将传入的负数转化为0。start可以大于end：

  • substr(start, [howmany])：从start处开始，截取howmany个字符返回。当传入的参数为负数时，会将负的第一个参数加上字符串的长度，而将负的第二个参数转化为0。

  • indexOf(str, [start])：从指定的位置开始从前向后查找字符串，start不可以为负数

  • lastIndexOf(str, [start])：从指定的位置开始从后往前查找字符串，同样start不可以为负数
  • trim()：去掉字符串的前后空格，返回新的字符串。不修改原来的字符串
  • toUpperCase()：将字符串中的字母转换为大写，原字符串保持不变（toLowerCase()与此相反）

  • match(reg)：将字符串与模式进行匹配，结果返回与reg.exec(str)相同的数组。注意：与reg.exec(str)和reg.test(str)等方法不同的是，使用str.match(reg)，在模式匹配到字符串时，RegExp对象上的lastIndex属性并不会得到更新，一直保持为 0 不变（包括在使用str.search(reg)时也是一样的情况），即使RegExp对象使用了全局匹配模式。

  • search(reg)：返回第一个匹配项的索引，每次都是从头开始匹配。如果没有找到匹配项，则返回 -1 。

  • replace(RegExp或string, string或function(match, group1, group2, ..., pos, input) {})：替换字符串。第一个参数可以传入RegExp对象或者字符串，而第二个参数传入的是字符串，可以在字符串中使用特殊的字符序列，比如：
    • $& 表示匹配整个模式的子字符串，与RegExp.lastMatch的值相同
    • $' 表示匹配的字符串之前的子字符串
    • $` 表示匹配的字符串之后的子字符串
    • $1，$2，… 表示模式中的捕获组匹配到的字符串
      replace方法可以接收一个函数作为第二个参数。该函数接收的参数是整个模式匹配到的字符串，第一个捕获组匹配到的字符串，第二个捕获组匹配到的字符串，…，匹配整个模式的字符串在原字符串中的位置以及原字符串的值。
  • split(RegExp或string, [arrSize])：分割字符串。分隔符可以是字符串，也可以是模式匹配到的字符串。不给出分隔符，则将该字符串作为数组仅有的一项返回。该方法还可以指定第二个参数，表示返回的数组不能超过指定的大小。（返回的数组长度可以小于指定的大小）

另外，在String构造函数上具有的静态方法

  • String.fromCharCode(code1, code2, ...)：返回由指定的字符编码对应的字符构成的字符串

String、Number和Boolean等基本包装类型在实际中其实很少使用，甚至要避免使用。更多时候，我们通过基本类型来使用其对应的基本包装类型对象上的方法。虽然有些方法看上去很简单，但也往往因此被我们忽略。希望我们都可以做一个有心人。
