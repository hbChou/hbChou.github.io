web的基础--HTTP协议

  • HTTP

HTTP协议，作为应用层的一种协议，是典型的B/S架构通信的基础。

协议版本

目前正在使用的HTTP协议都有以下几个版本：

  • HTTP/1.0是第一个得到广泛使用的HTTP版本。
  • HTTP/1.0+增加了一些新特性，包括持久的keep-alive连接、支持虚拟主机和代理连接。
  • HTTP/1.1是当前使用的HTTP版本。

HTTP报文组成

HTTP报文可以分成两种：请求报文和响应报文。报文一般由以下三部分组成：起始行、首部和主体。其中在首部和主体之间留有一个空行。

请求行

在请求行中，需要指定客户端希望服务器对资源执行的动作。常用的HTTP方法有：

  • GET：从服务器获取一份文档
  • POST：向服务器发送需要处理的数据
  • PUT：将请求的主体部分存储到服务器上
  • DELETE：从服务器删除一份文档。客户端无法保证删除操作一定会执行，因为HTTP规范允许服务器在不通知客户端的情况下撤销请求。
  • OPTIONS：询问服务器可以执行哪些方法
  • HEAD：只从服务器获取文档的首部。遵循HTTP/1.1规范，必须实现HEAD方法
  • TRACE：对可能经过代理服务器传送到服务器的报文进行追踪。行程中最后一站的服务器返回的响应主体中携带它收到的原始请求报文，以此和响应报文的首部比较，判断在请求/响应过程中请求报文是否被修改。

响应行

响应行中包含了响应报文使用的协议版本，响应状态码和响应状态说明信息。常见的状态码分成五类，分别是信息性状态码（100~199），成功状态码（200~299），重定向状态码（300~399），客户端错误状态码（400~499）和服务器端错误状态码（500~599）。

状态码原因短语含义
100Continue客户端在发送一个实体内容给服务器之前，希望查看一下服务器是否接受这个实体（客户端会发送一个携带了值为100 Continue的Expect请求首部）
200OK表示响应主体部分已经包含所请求的资源
301Moved Permanently在请求的资源被移除时使用，响应的Location首部中应该包含了资源现在所处的URL
302Found客户端应该使用Location首部给出的URL来临时定位资源，将来的请求仍应使用老的URL
304Not Modified表示资源未被修改，带有这个状态码的响应不应该包含实体主体部分
307Temporary Redirect与302状态码类似，客户端应该使用Location首部给出的URL来临时定位资源，将来的请求仍应使用老的URL
403Forbidden表示请求被服务器拒绝了
404Not Found表示服务器无法找到请求的URL

首部

HTTP报文中的首部可以分为四个主要的类型：

  • 通用首部 指的是客户端或服务器都可以使用的首部
  • 请求首部 这一部分是在请求报文中特有的首部
  • 响应首部 这一部分是在响应报文中特有的首部
  • 实体首部 用于说明实体主体部分的首部
    这里只列出常见首部。

通用首部

首部描述
Connection允许客户端和服务器端指定与连接相关的选项
Date提供日期和时间标志，说明报文是什么时候创建的
Transfer-Encoding告知接收端报文采用何种编码方式
Update给出了发送端可能想要升级使用的新版本或协议（不局限于HTTP协议）
Via显示了报文经过的中间节点（代理、网关）
Cache-Control随报文传送缓存指示

请求首部

首部描述
Host给出了接受请求的服务器的主机名和端口号
Referer提供了包含当前请求URL的文档的URL
User-Agent告知服务器发起请求的客户端应用程序名称
Accept告知服务器能够发送哪些媒体类型
Accept-Charset告知服务器能够发送哪些字符集
Accept-Encoding告知服务器能够发送哪些编码方式
Accept-Language告知服务器能够发送哪些语言
Expect允许客户端列出某请求所要求的服务器行为
If-Match如果实体标记与文档当前的实体标记相匹配，就获取这份文档
If-Modified-Since询问服务器所请求的URL指向的文档自从过去某个时间点以来是否发生修改
Cookie用于发送Cookie的首部

响应首部

首部描述
Age表示（从最初创建开始）响应持续时间
Public服务器为其资源提供的请求方法列表
Retry-After表示资源不可用时，在指定的日期和时间重试
Server服务器应用程序的名称和版本
Set-Cookie在客户端设置Cookie

实体首部

首部描述
Allow列出了可以对此实体执行的请求方法
Location用于将接收端定向到给定的URL上
Content-Base解析主体内容中的相对URL时使用的基础URL
Content-Length主体的长度
Content-Type主体的对象类型
ETag与此实体相关的实体标记
Expires实体不再有效，需要从服务器端再次获取此实体的日期和时间
Last-Modified实体最后一次被修改的日期和时间
