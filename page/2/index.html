浏览器是如何渲染页面的

  • 根据来自服务器端的HTML代码形成文档对象模型（DOM）
  • 加载并解析样式，形成CSS对象模型
  • 在文档对象模型和CSS对象模型之上，创建一棵由一组待生成渲染的对象组成的渲染树（在Webkit中这些对象被称为渲染器或渲染对象）。render tree能识别样式，上面的每个node都有自己的style，但是render tree中不包含诸如<head>标签或含有display:none属性的不可见元素。
  • 对渲染树上的每个元素，计算它的坐标，称之为布局。浏览器采用一种流方法，布局一个元素只需通过一次，但是表格元素需要通过多次。
  • 最后，渲染树上的元素最终展示在浏览器里，这一过程称为"painting"。当用户与网页交互，或者脚本程序改动修改网页时，前面提到的一些操作将会重复执行，因为网页的内在结构已经发生了改变。

回流与重绘

回流：当Render tree中的一部分(或全部)因为元素的规模尺寸，布局，隐藏等改变而需要重新构建。这就称为回流。每个页面至少需要一次回流，就是在页面第一次加载的时候。
重绘：当Render tree中的一些元素需要更新属性，而这些属性只是影响元素的外观，风格，而不会影响布局的，比如background-color，叫做重绘。

哪些操作会引起回流或重绘

其实任何对render tree中元素的操作都会引起回流或者重绘，比如：

  1. 添加、删除元素(回流+重绘)
  2. 隐藏元素，display:none(回流+重绘)，visibility:hidden(只重绘，不回流)
  3. 移动元素，比如改变top,left(重绘+回流)
  4. 对style的操作(对不同的属性操作，影响不一样)
  5. 还有一种是用户的操作，比如改变浏览器大小，改变浏览器的字体大小等(回流+重绘)
    如果每句JS操作都去回流重绘的话，浏览器可能就会受不了。所以很多浏览器都会优化这些操作，浏览器会维护一个队列，把所有会引起回流、重绘的操作放入这个队列，等队列中的操作到了一定的数量或者到了一定的时间间隔，浏览器就会清空队列，进行一个批处理。这样就会让多次的回流、重绘变成一次回流重绘。

如何减少回流或者重绘

  • 当需要设置多个样式时，设置节点的className属性而不是直接操作style属性
  • 让要操作的DOM元素进行”离线处理”，处理完后再一起更新，这里所谓的”离线处理”即让元素不存在于render tree中，比如在向页面添加多个新元素时使用documentFragment或div等元素进行缓存操作，最后再统一添加到页面中。

强制重排

要尽量避免回流和重绘，就要把访问元素属性的操作都组织在一起，从而优化网页性能

但有时，我们也必须触发一个强制性重排。比如，我们需要将同样的属性(比如左边距)两次赋值给同一个元素。起初，它应该设置为100px，且不带动效。接着，它必须通过过渡(transition)动效改变为50px。如果不强制重排，则动画不具备过渡效果。

雅虎军规

著名的前端优化方案有大名鼎鼎的“雅虎军规”。这里只列出其中的一些条目：

  • 压缩
    • 网页Gzip压缩
    • JS、CSS源码压缩
  • 缓存
    • 设置头文件过期或者静态缓存
    • 将JS和CSS外链

  • 性能

    • 避免CSS Expressions
    • 减少iframe数量
    • 缓存DOM节点查找、AJAX查询到的结果
    • 当需要设置的样式很多时候设置className而不是直接操作style属性
    • 用innerHTML代替DOM操作，减少DOM操作次数
    • 减少http请求次数：使用CSS Sprites，使用CDN托管（尽量使用公共库），避免重定向（比如在html中写入链接时，"github.io"和"github.io/"，前者会进一步重定向到后者）
    • js中要避免全局查找（可以将常用的全局变量作为参数传递到函数中），避免不必要的属性查找（可以将一次查找结果缓存在变量中），避免with语句

  • 用户体验

    • 图片预加载
    • CSS放顶部（让浏览者能尽早的看到网站的完整样式）、JS放底部（在不影响内容表现的情况下）
    • 尽早地释放缓冲（当用户进行页面请求时，服务器端需要花费200到500毫秒时间来拼合HTML，将写在head与body之间，释放缓冲，这样可以将文件头先发送出去，然后再发送文件内容，提高效率）
    • 跨域分离组件（页面组件多个来源可以增大你的平行下载量，但注意不要过多，超过2-4个域名会引起上面说到的DNS查找浪费）
    • 不要出现404页面

延伸扩展

  • 浏览器在解析域名的时候，是先从"."开始的，也就是最顶级的域名。

iframe的优缺点

iframe的优点：

  • 解决加载缓慢的第三方内容如图标和广告等的加载问题
  • Security sandbox（安全沙箱）
    • 禁止访问父页面的DOM（从技术角度来说，这是因为相对于父页面iframe已经成为不同的源了）
    • 禁止执行脚本
    • 禁止对cookie、本地存储或本地SQL数据库的读写
  • 并行加载脚本

缺点：

  • iframe会阻塞主页面的onload事件；
  • 即使内容为空，加载也需要时间

什么是CDN

CDN的全称是Content Delivery Network，内容分发网络。CDN公共库是指将常用的JS库存放在CDN节点，以方便广大开发者直接调用。与将JS库存放在服务器单机上相比，CDN公共库更加稳定、高速。使用了CDN缓存后的网站的访问过程为:

  • 用户向浏览器提供要访问的域名;
  • 浏览器调用域名解析库对域名进行解析，由于CDN对域名解析过程进行了调整，所以解析函数库一般得到的是该域名对应的CNAME记录，为了得到实际IP地址，浏览器需要再次对获得的CNAME域名进行解析以得到实际的IP地址;在此过程中，使用的全局负载均衡DNS解析，如根据地理位置信息解析对应的IP地址，使得用户能就近访问；
  • 此次解析得到CDN缓存服务器的IP地址，浏览器在得到实际的IP地址以后，向缓存服务器发出访问请求；
  • 缓存服务器根据浏览器提供的要访问的域名，通过Cache内部专用DNS解析得到此域名的实际IP地址，再由缓存服务器向此实际IP地址提交访问请求；
  • 缓存服务器从实际IP地址得得到内容以后，一方面在本地进行保存，以备以后使用，另一方面把获取的数据返回给客户端，完成数据服务过程；
  • 客户端得到由缓存服务器返回的数据以后显示出来并完成整个浏览的数据请求过程。

高效的css

浏览器在解析css选择器时是按从右向左的顺序匹配元素的。

原则

  • 避免使用通配符
  • 避免使用标签选择器和单个属性选择器作为关键选择器
  • 不要在id选择器前加标签名
  • 尽量不要在选择符定义过多层级，层级越少，同时也降低了css和dom结构的耦合程度，提高样式的可维护性

其他css高性能实践

  • 避免使用@import
    @import导入的新样式文件会阻止页面的并行下载，这样增加了文件的整体加载时间。
  • 避免使用IE浏览器独有样式：图片滤镜和css表达式
    图片滤镜的使用会在图片加载时阻塞浏览器的加载和渲染，并会增加额外的内存开销。Css表达式的作用是动态设置css属性，表达式不只是有兼容性问题，还有性能问题，例如浏览器大小改动、窗口改动时，会使得浏览器频繁计算，性能消耗极大。同样的效果可以用javascript来实现。

CSRF

什么是CSRF

CSRF：跨站请求伪造，核心也就是请求伪造，通过伪造身份提交POST和GET请求来进行跨域的攻击。
举一个通俗的例子：一论坛网站的发贴是通过 GET 请求访问，点击发贴之后 JS 把发贴内容拼接成目标 URL 并访问：
http://example.com/bbs/create_post.php?title=标题&content=内容
那么，我只需要在论坛中发一帖，包含一链接：
http://example.com/bbs/create_post.php?title=我是脑残&content=哈哈
只要有用户点击了这个链接，那么他们的帐户就会在不知情的情况下发布了这一帖子。可能这只是个恶作剧，但是既然发贴的请求可以伪造，那么删帖、转帐、改密码、发邮件全都可以伪造。

预防措施

首先可以提高的一个门槛，就是改良站内 API 的设计。对于发布帖子这一类创建资源的操作，应该只接受POST 请求，而 GET 请求应该只浏览而不改变服务器端资源。这样就把问题域缩小到了非 GET 类型的请求上——攻击者已经不可能通过发布链接来伪造请求了，但他们仍可以发布表单，或者在其他站点上使用我们肉眼不可见的表单，在后台用 js 操作，伪造请求。
对于POST请求，我们可以用比较简单也比较有效的方法来防御CSRF，这个方法就是“请求令牌”。“请求令牌”是为了保证收到的请求一定来自预期的页面。实现方法非常简单，首先服务器端要以某种策略生成随机字符串，作为令牌（token），保存在 Session 里。然后在发出请求的页面，把该令牌以隐藏域一类的形式，与其他信息一并发出。在接收请求的页面，把接收到的信息中的令牌与 Session 中的令牌比较，只有一致的时候才处理请求，否则返回 HTTP 403拒绝请求或者要求用户重新登录验证身份。
对于web站点，将持久化的授权方法（例如cookie或者HTTP授权）切换为瞬时的授权方法（在每个form中提供隐藏field），这将帮助网站防止这些攻击。

什么是跨域

javascript出于安全的考虑，不允许某个页面中的javascript代码直接操作来自不同域名的页面中的数据。两个页面使用的协议、域名和端口只要有一个存在不同，即认为是来自两个不同的域名，这时候双方的直接通信就存在着限制。有时候，我们就需要跟其他域名下的页面进行通信，这时候我们就需要了解javascript中都有哪些常用的跨域手段。

使用代理

我们可以在javascript中向同域名下的代理服务器发起请求，然后再由此代理服务器向目标服务器发起请求获取我们需要的数据。这是最直接有效的方法。

JSONP

JSONP只能用于GET请求中。它的基本思想是客户端通过请求参数向服务器端传送一个用于处理数据的回调函数的名称，由服务器向客户端返回一个字符串，该字符串类似于回调函数调用的形式，将要返回给客户端的数据作为参数传入该回调函数中。客户端在接收到字符串之后，创建script标签，将字符串作为script标签的内容并添加到页面中。这时候就实现将数据传入到客户端预先定义好的回调函数中。接着就在回调函数中处理服务器端返回的数据。
另外一种做法是直接为script标签的src属性指定一个跨域的URL，利用script标签不受限制地从其他域加载资源的能力。服务器端动态地创建客户端所请求的javascript文件并返回给客户端，客户端将文件加载到页面中即可立即执行代码，获取到所需要的数据。

jQuery中实现JSONP

JSONP存在的不足：只能用于GET请求中。存在一定的安全问题，如果其他域不安全，则有可能会在响应中夹杂一些恶意代码。难以确定JSONP请求是否失败。

跨域资源共享（CORS）

跨域资源共享（CORS）定义了浏览器和服务器如何通过可控方式进行跨域通信。CORS通过添加特殊HTTP头信息以允许浏览器和服务器判断请求是成功还是失败。几乎所有现代浏览器都支持CORS。

HTTP请求头和响应头

当跨域发送HTTP请求时，支持CORS的浏览器会通过添加额外Origin头信息指定请求源，其值包括请求协议、域名、端口。如：
Origin: http://www.codingplayboy.com
若请求头中无Origin信息，服务器将不返回任何CORS头信息。

服务端接收到请求时会检查头信息确定是否接受请求：若接受请求，必须返回一个包含Access-Control-Allow-Origin响应头，其值与请求头Origin值相同，如：
Access-Control-Allow-Origin: http://www.codingplayboy.com

创建CORS请求

IE对CORS的实现

IE8引入了XDR（XDomainRequest）对象，该对象和XHR对象类似，但能实现安全的跨域通信。XDR对象本身具有的一些安全策略：

  • cookie不会随请求发送，也不会随响应返回
  • 只能设置请求头中的Content-Type字段
  • 不能访问响应头信息
  • 只支持GET和POST请求
    XDR的使用方法和XHR非常类似，不同的是，XDR的open方法只接收两个参数，分别是请求方法和请求的URL。所有XDR的请求都是异步的，不能用它来创建同步请求。服务器端返回响应之后，会触发load事件，响应的数据也保存在XDR对象的responseText属性中。只能访问响应主体部分的数据，没有办法确定响应的状态码。也可以通过XDR对象发送POST请求，只需要修改请求头中的Content-Type字段（xdr.contentType），将要发送的数据传入send方法即可。

其他浏览器中的CORS实现

与IE中的XDR对象不同，XHR对象可以访问status和statusText属性，而且还支持同步请求。同样其他浏览器中的XHR对象也存在一些安全策略：

  • 不能使用setRequestHeader()方法设置自定义首部
  • 不能发送和接收cookie
  • 不能访问响应头信息
    默认情况下，跨域请求不提供凭据（cookie，HTTP认证及客户端SSL证明等）。可以通过将XHR对象上的withCredentials属性设置为true，可以指定某个请求发送凭据。如果服务器接受带凭据的请求，则会用下面的HTTP头部来响应：
    Access-Control-Allow-Credentials: true
    如果发送带凭据的请求，但服务器的响应中没有包含这个头部，则浏览器不会把响应交给javascript。（responseText将是空字符串，status值为0，而且会调用onerror事件处理程序）

预检请求（Preflight Requests）

使用CORS时，若请求方法不是GET，POST或HEAD，又或者使用了自定义HTTP头，浏览器将发起预检请求。预检请求是一个服务端认证机制，在执行请求之前会判断该请求是否合法。发送复杂请求时，浏览器会将原始请求的方法和请求头作为预检请求的信息发送给服务器；服务器需要决定是否接受该请求并响应，预检请求通常是使用一种OPTIONS的HTTP方法。

客户端发起复杂请求时，会先发起预检，携带以下头信息:

请求头描述
Origin请求源
Access-Control-Request-Method请求方法（HTTP方法）
Access-Control-Request-Headers请求自定义头（以逗号分隔）
服务器返回的响应头：
响应头描述
Access-Control-Allow-Origin允许的请求源（与请求头Origin匹配）
Access-Control-Allow-Methods允许的请求方法（以逗号分隔）
Access-Control-Allow-Headers允许的头信息（以逗号分隔）
Access-Control-Max-Age预检请求的缓存时间（单位：秒）
Access-Control-Allow-Credentials指定请求是否支持认证信息（可选）
客户端接收到服务端的响应后，会使用之前声明的HTTP方法和请求头发送真正的请求。预检请求会被浏览器缓存，缓存时间为Access-Control-Max-Age值，缓存时间内，后续相同类型的请求不再发起重复的预检请求。IE8、IE9均不支持预检请求。

HTML5中的XDM（cross-document messaging，跨文档消息传送）

XDM的核心方法是postMessage方法。HTML5的window.postMessage为浏览器带来了一个安全的、基于事件的消息API。与之前的子域名代理通过iframe跨子域通信不同，使用postMessage不再是直接访问一个文档的属性和方法，而是向文档发送消息然后等待响应，这要求形成一条双向的通信通道。
对于XDM而言，传送消息的目的地指的是包含在当前页面中的iframe元素，或者由当前页面弹出的窗口。postMessage()方法接收两个参数：一条消息和一个表示消息接收方来自哪个域的字符串。这样可以防止浏览器把消息发送到不安全的地方去。

如果指定的域名和内嵌框架加载的页面的域名一致，则会将消息传递到内嵌框架中。接收方在接收到消息时，会触发window对象上的message事件，传递到onmessage事件处理程序中的event对象包含以下三个重要信息：

  • data：接收到的消息
  • origin：消息发送方所在的域
  • source：发送消息的文档的window对象的代理。这个代理对象主要调用postMessage方法，向发送方发送回执消息。
    通常在onmessage事件处理程序中需要通过event.origin检测消息的来源。这样可以确保传入的消息来自已知的页面。

选择符API

选择符API level 1的核心是两个方法：querySelector(selector)和querySelectorAll(selector)
querySelector(selector)方法接收一个css选择符，返回符合该选择符的第一个DOM元素
既可以通过document对象调用该方法，也可以直接在某个元素节点或者文档片段节点上调用。
querySelector(selector)方法接收一个css选择符，返回由符合该选择符的所有DOM元素组成的NodeList对象。需要特别注意的是，该NodeList对象和之前的document.getElementsByTagName(tagName)方法返回的NodeList对象略有差别，前者是创建该对象瞬间符合该选择器的所有DOM元素集合的一个快照，此后不会随着DOM结构的变化而发生任何更新。而后者则刚好相反。同样既可以通过document对象调用该方法，也可以直接在某个元素节点或者文档片段节点上调用。

选择符API level 2中为Element类型新增了一个方法matchesSelector()，用于判断某个元素是否符合给定的选择符。该方法返回一个布尔值。该方法在不同的浏览器中有一些实验性的实现。比如在chrome浏览器中通过webkitMatchesSelector()支持该方法，在firefox中则通过mozMatchesSelector()实现了该方法。

由于IE和其他浏览器在识别节点内部的换行符时存在差异，为此，Element Traversal规范新定义了一组属性：

  • childElementCount：返回子元素（元素节点，不包括文本节点和注释）的个数
  • firstElementChild：指向第一个子元素
  • lastElementChild：指向最后一个子元素
  • previousElementSibling：指向前一个同胞元素
  • nextElementSibling：指向后一个同胞元素

HTML5中与class属性相关的扩充

getElementsByClassName("className1 className2 ...")：返回由包含指定类名的所有元素构成的NodeList对象。可以同时传递多个类名。该方法返回的NodeList对象与getElementsByTagName()返回的NodeList对象一样存在着性能问题。（实时更新内部的DOM元素信息）

HTML5为所有元素节点添加了classList属性，以简化原生javascript直接添加，删除和修改类名的操作。该属性指向一个DOMTokenList对象，里面包含了元素包含的所有类名。取得每个元素可以使用item()方法，也可以使用方括号语法。该对象上还定义了一些方法：

  • add(value)：将给定的字符串添加到class属性值列表中，如果存在，则不添加
  • contains(value)：表示class属性值列表中是否存在该字符串，返回一个布尔值
  • remove(value)：从class属性值列表中移除给定的字符串
  • toggle(value)：如果列表中已经存在给定的字符串，则删除它，否则添加该字符串
    支持classList属性的浏览器由Firefox 3.6+、chrome和IE 10+。

管理DOM焦点

HTML5中也添加了辅助管理DOM焦点的功能。添加的document.activeElement属性始终会引用DOM中当前获得焦点的元素。元素获得焦点的方式有页面加载、用户输入和在代码中调用focus()方法。默认情况下，文档刚加载完成时，document.activeElement属性中保存的是对document.body的引用，在加载过程中document.activeElement属性值为null。
另外新增了document.hasFocus()方法，用于判断当前文档是否获得焦点。

HTMLDocument的变化

  • readyState属性
    IE4最早为document对象添加了readyState属性。该属性有两个可能的值：

    • "loading"：表示正在加载文档；
    • "complete"：表示已经加载完文档
      通过该属性可以实现类似window.onload事件的用来指示文档加载完毕的指示器。

  • 兼容模式
    document.compatMode属性用来检测当前浏览器渲染页面的模式是标准的还是混杂的。在标准模式下，document.compatMode属性值为"CSS1Compat"，在混杂模式下，document.compatMode属性值为"BackCompat"。

  • head属性
    通过document.head属性快速获取到页面的head元素，实现该属性的浏览器有chrome、safari 5和IE 9+。

  • 字符集属性
    通过document.charset属性获取或修改当前文档实际使用的字符集。默认情况下该属性值为"UTF-16"。另外还有一个属性document.defaultCharset表示根据浏览器和操作系统的默认设置，当前文档默认的字符集应该是什么。
    支持document.charset属性的浏览器有IE、safari、Opera、chrome和Firefox。支持document.defaultCharset属性的浏览器有IE、safari和chrome。

自定义数据属性

添加的自定义属性名称必须以"data-"为前缀，才能被元素节点的dataset属性来访问自定义属性的值。dataset属性的值是一个DOMStringMap的一个实例，也就是一个键值对的映射。支持自定义属性的浏览器有Firefox 6、chrome和IE 11+。

innerHTML属性

大多数浏览器中使用innerHTML属性插入script标签时，内部的javascript代码是不会执行的。而在IE9及以下版本中需要一些特殊条件才能执行。一是必须为script标签指定defer属性，二是script标签必须位于有作用域的元素之后才能起作用，所谓的有作用域元素可以是一个文本节点，也可以是一个没有结束标签的input元素。通常在要插入的script前面插入type属性值为hidden的input元素。

与插入script标签的情况相反，在大多数浏览器中使用innerHTML属性插入style标签是有效的。但在IE8及以下版本中还是需要在style标签前面插入一个有作用域的元素。

最后，并不是所有的元素都支持innerHTML属性。不支持innerHTML属性的元素有<head>、<html>、<style>、<table>、<tbody>、<thead>、<tfoot>和<tr>。另外在使用innerHTML属性插入html代码（尤其是来自用户输入的内容）时，需要先进行HTML编码再进行插入。

通过元素节点的outerHTML属性可以访问元素节点本身及其所有子节点的HTML标签。修改该属性会直接替换该元素以及所有子节点。

DOM中的节点

DOM可以将任何HTML或XML文档描述成一个多层节点构成的树状结构。文档节点（在HTML中即我们常用的document对象）是整个文档的根节点。对于HTML或者XML文档，文档节点通常包含一个文档元素，文档元素是文档的最外层元素，文档中的其他所有元素都包含在文档元素中。在一般的HTML文档中，文档节点通常只包含两个子节点，即文档类型节点（<!DOCTYPE html>）和文档元素（即html元素）。而在XML文档中，任何元素都可能成为文档元素。

Node类型

javascript中的所有节点类型都继承Node类型。除了IE之外，其他浏览器中都可以访问到Node这个接口。
每个节点都一个nodeType属性，表示元素的类型。常见的元素类型（后面的数值表示类型对应的数值常量）有：

  • Node.ELEMENT_NODE（元素节点） 1
  • Node.ATTRIBUTE_NODE（属性节点：在文档树中不作为单独的节点出现） 2
  • Node.TEXT_NODE（文本节点：元素中包含的文字以及元素之间的空白均属于文本节点） 3
  • Node.COMMENT_NODE（注释节点） 8
  • Node.DOCUMENT_NODE（文档节点：文档的根节点，根节点不等同于根元素，例如根元素（如html元素）外部可以有注释节点等其他节点） 9
  • Node.DOCUMENT_TYPE_NODE（文档类型节点：在HTML文档中的第一行<!DOCTYPE html>就属于文档类型节点） 10
  • Node.DOCUMENT_FRAGMENT_NODE（文档片段节点：不属于文档树） 11
    注意：在判断某个节点的类型时，常将该节点的nodeType属性和对应的数值常量进行比较，才能保证实现跨浏览器。这是因为IE浏览器没有公开Node类型的构造函数。

此外每个节点还包含一个nodeName和nodeValue属性。
nodeName属性：用于获取节点的名称。对于元素节点，nodeName代表的是元素的标签名称；对于属性节点，nodeName代表的是属性名称；对于文本节点，nodeName是"#text"；对于注释节点，nodeName是"#comment"。获取文档类型可以使用document.doctype，在HTML文档中该节点的nodeName为"html"，对于文档片段节点，nodeName为`"#document-fragment"。
nodeValue属性：用于获取节点的值；对于元素节点，nodeValue代表的是null；对于属性节点，nodeValue 返回的是属性值；对于文本节点，nodeValue返回的是节点所包含的文本；对于注释节点，nodeValue返回的是注释内容；对于文档类型节点，nodeValue的值为null；对于文档片段节点，nodeValue为null。

节点具有的一般属性和方法

每个节点都有一个childNodes属性，该属性保存着一个NodeList对象，该对象是一个类数组对象。（如果该节点没有子节点，则返回一个长度为0的NodeList对象，而不是null）访问NodeList中的节点，可以使用方括号的形式，也可以使用该对象上的item方法。注意：NodeList对象中的节点会随着DOM结构中节点的变化而实时更新，不是第一次创建它们时那一瞬间拍下来的一张快照。

在除IE之外的其他主流浏览器中，节点内部的换行符会被看成是一个文本节点。这意味着在执行某项操作时，需要特别留意由节点中间的换行符形成的文本节点，通常需要先判断节点的nodeType属性。

每个节点都有一个parentNode属性，该属性指向当前节点的父节点。parentNode属性指向同一对象的所有为同胞节点，可以通过previousSibling和nextSibling属性访问节点的其他同胞节点。
包含子节点的节点可以通过firstChild和lastChild属性分别访问该节点的第一个子节点和最后一个子节点。而对于不确定是否包含子节点的节点可以通过hasChildNodes()方法来判断是否包含子节点，该方法在节点包含一个或多个子节点的情况下返回true。
每个节点都有一个ownerDocument属性，该属性指向整个文档的文档节点。

每个节点都一个cloneNode()方法，该方法用于创建并返回跟调用此方法的节点相同的一个副本。该方法接收一个布尔值参数，表示是否执行深复制。在参数为true的情况下，执行深复制，也就是复制该节点及其以下的整个子节点树。而执行浅复制时，则只复制该节点本身，内部的子节点不会被复制，也就是说，复制得到的节点只有一个，该节点不包含子节点。
注意：该方法不会复制添加到DOM节点中的事件处理程序。

javascript中一些操作节点的方法

创建元素节点：document.createElement(tagName)：该方法返回一个新创建元素节点的引用。
创建文本节点：document.createTextNode(string)：该方法返回一个新创建文本节点的引用。
创建注释节点：document.createComment(string)：该方法返回一个新创建注释节点的引用。
appendChild(newNode)方法用于向指定节点的尾部插入新节点。更新完成后，该方法返回新增的节点。
如果传入的新节点已经是当前文档的一部分了，则会将该新节点从原来的位置转移到指定的节点的尾部。
如果需要将节点插入到另一个节点中特定的位置，则可以使用insertBefore(newNode, referenceNode)方法。该方法接受两个参数：要插入的节点和作为参照的节点。同样返回新插入的节点。
替换节点：replaceChild(newNode, referenceNode)，该方法接受两个参数：要插入的节点和要被替换的节点。返回被替换掉的节点。
移除节点：removeChild(Node)，该方法接收一个参数：即要被移除的节点。返回被移除的节点。
以上这四个方法操作的都是某个节点的子节点，因此要使用这些方法，需要先获取到要操作的这些节点的父节点。

Document类型

document对象上的常用属性

document.documentElement：指向当前html文档的文档元素
document.body：指向当前html文档的body元素
document.doctype：指向当前html文档的文档类型节点（在IE8及以下版本为null，而其他浏览器在关于文档类型节点属不属于文档的子节点的问题上有较大差异）
document.doctype.name：返回文档类型的名称
document.title：包含title元素中的文本
document.URL：包含当前页面完整的URL
document.domain：包含当前页面所在的域名，该属性可以设置。但不能将该属性设置为URL中不包含的域名。此外还有一个限制，document.domain设置的值不能比原有的值范围更窄。比如document.domain的值原本是github.io，就不能再把它设置成xxx.github.io。
由于同源策略的限制，来自不同子域的页面无法通过javascript进行通信。这时候可以在不同的子域下的页面中将document.domain设置成相同的值，然后在主页面中通过内嵌框架加载需要进行通信的页面，就能够实现来自不同子域的页面进行通信。
document.referrer：包含链接到当前页面的另一个页面的URL

通过document对象获取元素

document.getElementById(id)：查找具有指定id属性值的元素。如果不存在该元素，则返回null。
document.getElementsByTagName(tagName)：查找具有指定标签名的元素集合。结果返回包含零个或多个元素的HTMLCollection对象。HTMLCollection对象和NodeList对象类似。可以通过方括号或者item方法访问集合中的元素。此外，HTMLCollection对象上还有一个namedItem(name)方法，该方法接收一个元素的name属性值，返回集合中具有指定name属性的元素。
document.getElementsByName(name)：返回带有指定name属性的所有元素。该方法也会返回一个HTMLCollection对象。
以下的document对象属性同样也会返回HTMLCollection对象：

  • document.anchors：该属性会返回页面中所有带有name属性的a标签
  • document.forms：该属性会返回页面中所有的form元素
  • document.images：该属性会返回页面中所有的image元素
  • doucment.links：该属性会返回页面中带有href属性的所有a元素

文档写入

document.write(string)和document.writeln(string)：向页面动态添加内容，使用document.writeln(string)则会在string参数后面自动添加换行符（"\n"）。在页面加载的过程中使用这两个方法可以直接向页面输出内容，而如果是在页面加载结束调用这两个方法，则输出的内容会重写整个页面的内容。

Element类型

访问元素的标签名，除了可以通过元素的nodeName属性访问，还可以通过tagName属性。注意：两者返回的标签名都是大写的。
访问元素的class属性，需要通过元素的className属性来访问，因为class是javascript中保留字。

操作元素的属性

获取元素的属性：getAttribute(attr)，此时访问元素的class属性，需要传入"class"而不是"className"。如果要访问的属性不存在，则返回null。
除了自定义属性之外，直接通过HTMLElement对象上的属性就可以访问到定义在元素标签上的属性值。而通过getAttribute(attr)方法可以访问定义在元素标签上的所有属性（包括自定义属性）。但它直接返回元素标签上对应的属性值（该值为字符串），导致在访问一些属性时返回的值与直接访问HTMLElement对象上的属性得到的值有所差异。
设置元素的属性：setAttribute(attr, value)，
移除元素的属性：removeAttribute(attr)
这三个方法可以认为是直接操作元素标签上的属性和值。

元素节点的attributes属性

Element类型对象上包含了attributes属性，该属性指向一个NamedNodeMap对象，该对象和NodeList对象类似。元素标签上定义的每一个属性都有一个Attr节点表示，一个元素的所有Attr节点就保存在一个NamedNodeMap对象中。NamedNodeMap对象拥有的方法：
getNamedItem(name)：返回nodeName属性等于name的Attr节点
removeNamedItem(name)：从Attr节点列表中移除nodeName属性等于name的节点并返回该节点
setNamedItem(Attr)：向Attr节点列表中添加新的Attr节点
item(pos)：返回索引为pos对应的Attr节点
利用attributes属性来访问元素的属性比直接使用getAttribute(attr)等方法要繁琐的多，实际用到的场景也不多，不过用它来遍历元素的属性还是很方便的。

Text类型

文本节点用Text类型来表示。可以通过nodeValue属性或data属性来访问文本节点中的内容，两者返回的值是一样的。也可以通过这两个属性来修改文本节点里面的内容。

一些操作文本节点内容的属性和方法

  • length：保存着文本节点中字符的个数
  • appendData(text)：将text添加到节点的末尾
  • deleteData(offset, count)：从offset指定的位置开始删除count个字符
  • insertData(offset, text)：在offset指定的位置插入text
  • replaceData(offset, count, text)：用text替换从offset指定的位置开始到offset``+``count为止处的文本
  • splitText(offset)：从offset指定的位置将文本节点分成两个文本节点。原来的文本节点保存从0到指定位置处的文本内容，然后创建一个新的包含剩下文本的文本节点并返回。
  • substringData(offset, count)：提取从offset指定的位置开始到offset``+``count为止处字符串
    注意：通过文本节点添加的内容里面的特殊字符（如< >）会被自动进行HTML编码。

多个连续文本节点的合并

在DOM操作中，有时会面临同一节点内部存在着多个连续的文本节点，这容易造成混乱，让人无法识别哪个文本节点表示哪个字符串。此时在父节点上调用normalize()方法可以将多个连续的文本节点合并成一个文本节点。

DocumentFragment类型

DocumentFragment类型的对象常用来存放待添加到文档中的节点。
创建文档片段的方法：document.createDocumentFragment()
如果将文档中的节点添加到文档片段中时，就会从文档树中移除该节点。可以使用前面的通用的DOM操作方法（appendChild()、insertBefore()等方法）将节点添加到文档节点中或者将文档节点添加到文档树中。将文档片段添加到文档树中时，只有内部的子节点会添加到文档树中，文档片段本身不会添加到文档树中。

Attr类型

元素标签上定义的属性在DOM中用Attr类型来表示。
创建属性节点：document.createAttribute(name)，该方法返回一个属性节点的引用，另外还需要为该属性节点的value属性指定属性节点的值
Attr对象有两个常用的属性：

  • name：元素标签上定义的属性名称
  • value：元素标签上定义的属性的值
    将新创建出来的属性节点添加到元素中，还必须调用元素节点的setAttributeNode(AttrNode)方法，传入新创建的属性节点即可。

window对象

全局作用域中的变量

我们都知道在全局作用域中定义一个变量，都会成为window对象上的一个属性。在大部分情况下，它们两者是相同的。但是在对两者使用delete操作符时还是有区别的。

javascript中使用var声明变量作为属性添加到window对象上时，该属性的configurable特性默认设置为false，因此在使用delete删除属性时会返回false。而在IE9以下版本删除window对象上的属性都会报错，无论该属性是通过var声明变量添加的还是在window对象上直接添加的。

window对象上那些和框架相关的属性

window.top 对象始终指向最外层的框架
window.parent 对象指向当前框架的直接上层框架
window.self 对象指向当前的window对象（和window.window属性指向同一个对象）

窗口位置和大小

window.screenLeft和window.screenTop（在firefox中使用window.screenX和window.screenY）：表示窗口相对于屏幕左边和上边的距离。在不同的浏览器中它表示的含义还有些不同。比如在IE、Opera中，screenLeft和screenTop中保存的是从屏幕左边和上边到页面可见区域的距离，而在chrome、safari和firefox中，这两个属性保存的时整个浏览器窗口相对于屏幕左边和上边的距离。
window.innerWidth和window.innerHeight：表示页面视图的宽高
window.outerWidth和window.outerHeight：表示浏览器窗口本身的宽高
在IE、firefox、chrome、safari和opera中，document.documentElement.clientWidth和document.documentElement.clientHeight保存了页面视口的宽高（这两个属性在IE6中仅在标准模式下有效），在混杂模式下使用document.body.clientWidth和document.body.clientHeight。

打开新窗口

window.open()方法接收四个参数：要加载的url、窗口目标（已有窗口或框架的名称，在指定的窗口或框架中加载新页面）、一个特性字符串以及一个表示新页面是否在浏览器页面历史中代替当前页面的布尔值（该值仅在不打开新窗口的情况下使用）。
第二个参数除了可以指定现有窗口或框架名称，也可以是以下的字符串之一：
_self：在当前窗口显示目标网页
_parent：在当前框架的直接上层框架中显示目标网页
_top：在最外层的框架中显示目标网页
_blank：在新窗口中显示目标网页
第三个参数可以设置的新窗口选项：
width：数值，height：数值，
left：数值，top：数值，left和top分别指定打开的窗口距离屏幕左边和上边的距离，
resizable：yes或no
status：yes或no（是否显示状态栏），toolbar：yes或no（是否显示工具栏），
scrollbars：yes或no（是否显示滚动条）
window.open()方法会返回一个对新窗口的引用。例如

可以通过window.close()来关闭弹出窗口。
新建的window对象有一个opener属性，保存着打开它的原始窗口对象。

location对象

location对象提供了当前窗口中加载的文档有关的信息，还将URL解析成独立的片段供我们使用。它既是window对象的属性，也是document的属性。

location对象的常用属性

hash 返回url中的hash（即#号后面跟零或多个字符）
host 返回服务器主机的域名和端口号
hostname 返回服务器主机的域名（不带端口号）
href 返回当前页面的完整url（location对象的toString方法也返回这个值）
pathname 返回当前url中的路径信息
port 返回当前url中的端口号
protocol 返回当前url所使用的协议
search 返回当前url的查询参数（如果有的话，返回的字符串以问号开头）

通过location对象跟浏览器位置有关的方法

  • 使用location对象的assign方法
    该方法接收一个url字符串，该方法立即打开新的页面并在浏览器的历史记录中生成一条记录。此外，修改location.href或window.location的值，也会调用location.assign方法。
  • 使用location对象的replace方法
    该方法也是接收一个url字符串，该方法会导致浏览器位置改变，但不会浏览器的历史记录中生成一条记录，也就是说，用户无法通过后退按钮返回前一个页面。
  • 使用location对象的reload方法
    该方法用于重新加载当前页面，它会选择最有效的方式来加载。默认情况下，如果当前页面自上次请求以来没有被修改，则会从浏览器的缓存中加载当前页面。如果要强制从服务器重新加载页面，则可以给该方法传递一个布尔值：true。

history对象

history对象保存着用户上网的历史记录。每个浏览器窗口、每个标签页以及每个框架都有自己的history对象和特定的window对象相关联。

history对象的常用方法和属性

history.go()：该方法接收一个数值，负数表示向后跳转的页面数，而正数则表示向前跳转。该方法也可以接收一个字符串参数，表示跳转到历史记录中包含指定字符串的记录对应的第一个页面（有可能向前跳转也有可能向后）
history.back()：模仿浏览器上的后退按钮
history.forward()：模仿浏览器上的前进按钮
history.length：保存着历史记录的数量

navigator对象

navigator对象主要用于识别客户端浏览器，比如它有一个常用的属性：navigator.userAgent。该属性返回用户代理头的字符串表示(就是包括浏览器版本信息等的字符串)
