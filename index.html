web的基础--HTTP协议

HTTP协议，作为应用层的一种协议，是典型的B/S架构通信的基础。

协议版本

目前正在使用的HTTP协议都有以下几个版本：

  • HTTP/1.0是第一个得到广泛使用的HTTP版本。
  • HTTP/1.0+增加了一些新特性，包括持久的keep-alive连接、支持虚拟主机和代理连接。
  • HTTP/1.1是当前使用的HTTP版本。

HTTP报文组成

HTTP报文可以分成两种：请求报文和响应报文。报文一般由以下三部分组成：起始行、首部和主体。其中在首部和主体之间留有一个空行。

请求行

在请求行中，需要指定客户端希望服务器对资源执行的动作。常用的HTTP方法有：

  • GET：从服务器获取一份文档
  • POST：向服务器发送需要处理的数据
  • PUT：将请求的主体部分存储到服务器上
  • DELETE：从服务器删除一份文档。客户端无法保证删除操作一定会执行，因为HTTP规范允许服务器在不通知客户端的情况下撤销请求。
  • OPTIONS：询问服务器可以执行哪些方法
  • HEAD：只从服务器获取文档的首部。遵循HTTP/1.1规范，必须实现HEAD方法
  • TRACE：对可能经过代理服务器传送到服务器的报文进行追踪。行程中最后一站的服务器返回的响应主体中携带它收到的原始请求报文，以此和响应报文的首部比较，判断在请求/响应过程中请求报文是否被修改。

响应行

响应行中包含了响应报文使用的协议版本，响应状态码和响应状态说明信息。常见的状态码分成五类，分别是信息性状态码（100~199），成功状态码（200~299），重定向状态码（300~399），客户端错误状态码（400~499）和服务器端错误状态码（500~599）。

状态码原因短语含义
100Continue客户端在发送一个实体内容给服务器之前，希望查看一下服务器是否接受这个实体（客户端会发送一个携带了值为100 Continue的Expect请求首部）
200OK表示响应主体部分已经包含所请求的资源
301Moved Permanently在请求的资源被移除时使用，响应的Location首部中应该包含了资源现在所处的URL
302Found客户端应该使用Location首部给出的URL来临时定位资源，将来的请求仍应使用老的URL
304Not Modified表示资源未被修改，带有这个状态码的响应不应该包含实体主体部分
307Temporary Redirect与302状态码类似，客户端应该使用Location首部给出的URL来临时定位资源，将来的请求仍应使用老的URL
403Forbidden表示请求被服务器拒绝了
404Not Found表示服务器无法找到请求的URL

首部

HTTP报文中的首部可以分为四个主要的类型：

  • 通用首部 指的是客户端或服务器都可以使用的首部
  • 请求首部 这一部分是在请求报文中特有的首部
  • 响应首部 这一部分是在响应报文中特有的首部
  • 实体首部 用于说明实体主体部分的首部
    这里只列出常见首部。

通用首部

首部描述

你所不知道的javascript之Ajax的使用

第一次接触Ajax时，就了解到它可以做到页面局部内容的动态刷新，而不需要从服务器端请求整个页面。这对于页面中一些只需要请求少量数据的场景，使用它是再合适不过了。Ajax的核心是XMLHttpRequest这个对象。在javascript中，直接使用new关键字创建该对象即可。

XHR对象上的一些常用属性和方法

在发送请求时，可以使用的方法：

  • open方法：该方法接收三个参数：请求的方法，请求的URL和是否异步发送请求的布尔值。
    受同源策略的限制，在默认情况下，只能通过Ajax向同域名下的主机（相同协议、域名和端口）发送请求。
  • setRequestHeader方法：该方法接收两个参数：头部字段名称和要设置的值。该方法必须在open方法和send方法之间调用。一般需要手动修改的请求字段是”Content-Type”这个字段。发送post请求时一般需要将这个字段的值设置为”application/x-www-form-urlencoded”。
  • send方法：该方法接收一个可选的参数：在发送post请求时要发送的请求数据。要发送的数据需要首先进行序列化，然后再进行发送。最简单的方法是通过FormData对象来序列化表单。传入的数据也可以是普通的字符串。
    在收到响应时，可以使用的属性和方法：
  • responseText：响应主体内容
  • responseXML：如果响应到的内容类型是”text/xml”或”application/xml”，则该属性保存着响应的数据主体内容
  • status：响应的状态码
  • statusText：响应状态码的说明信息
  • getResponseHeader方法：该方法接收一个参数：即要获取的响应头部字段的名称
  • getAllResponseHeaders方法：返回包含所有响应首部信息的字符串
    在发送异步请求时，常需要用到XHR对象上的onreadystatechange事件，该事件会在XHR对象的readyState属性值发生变化的时候触发，通过readyState属性的值可以得知当前发送异步请求所处的状态，该属性有5个可能的值：
  • 0：未初始化，即未调用XHR对象的open方法
  • 1：启动，已经调用open方法，而还没有调用send方法
  • 2：发送，已经调用send方法，但还没有收到响应数据
  • 3：接收，已经接收到部分响应数据
  • 4：完成，已经接收到全部的响应数据

unquote($string)：删除字符串中的引号；

To-upper-case() 函数将字符串小写字母转换成大写字母（与此相反的还有To-lower-case()）

数字函数

percentage($value)：将一个不带单位的数转换成百分比值；如果转换的值是一个带有单位的值，那么在编译的时候会报错误信息
round($value)：将数值四舍五入，转换成一个最接近的整数；参数可以是携带任意单位的数值
ceil($value)：将大于自己的小数转换成下一位整数；
floor($value)：将一个数去除他的小数部分；
abs($value)：返回一个数的绝对值；
min($numbers…)：找出几个数值之间的最小值；在 min() 函数中同时出现两种不同类型的单位，将会报错
max($numbers…)：找出几个数值之间的最大值；
random(): 获取随机数

列表函数

length($list)：返回一个列表的长度值；length()函数中的列表参数之间使用空格隔开，不能使用逗号，否则函数将会出错。例如：

nth($list, $n)：返回一个列表中指定的某个标签值，$n 必须是大于 0 的整数。

join($list1, $list2, [$separator])：将两个列给连接在一起，变成一个列表；只能将两个列表连接成一个列表，如果直接连接两个以上的列表将会报错。参数 $separator主要是用来给列表函数连接列表值时使用的分隔符号，默认值为 auto。
join() 函数中 $separator 除了默认值 auto 之外，还有 comma 和 space 两个值，其中 comma 值指定列表中的列表项值之间使用逗号（","）分隔，space 值指定列表中的列表项值之间使用空格（" "）分隔。不明确指定分隔符时，会出现以下情况：
如果列表中的第一个列表中有多个值，那么 join() 函数合并的列表中每个列表项之间使用第一个列表中的分隔符进行分隔；但当第一个列表中只有一个列表项，那么 join() 函数合并的列表项目中每个列表项目之间将会使用第二个列表项使用的分隔符。如果两个列表都只有一项，则合成的列表中每一项使用空格进行分隔。

append($list1, $val, [$separator])：将某个值放在列表的最后；在不明确指定分隔符时，如果列表只有一个列表项，那么插入进来的值将和原来的值以空格的方式分隔。

zip($lists…)：将几个列表结合成一个多维的列表。例如：

index($list, $value)：返回一个值在列表中的位置值。如果指定的值不在列表中（没有找到相应的值），那么返回的值将是 false，否则会返回对应的值在列表中所处的位置。

SASS中的数据类型

字符串

字符串有两种：
有引号字符串(quoted strings)，如 "Lucida Grande" 、'http://sass-lang.com'；
无引号字符串(unquoted strings)，如 sans-serifbold。
在编译 CSS 文件时不会改变其类型。只有一种情况例外，使用 #{}插值语句时，有引号字符串将被编译为无引号字符串，这样方便了在混合指令 (mixin) 中引用选择器名。

值列表

通过空格或者逗号分隔的一系列的值。
值列表函数

  1. nth函数可以直接访问值列表中的某一项；
  2. join函数可以将多个值列表连结在一起；
  3. append函数可以在值列表中添加值；
  4. @each规则能够给值列表中的每个项目添加样式。
    值列表中可以再包含值列表，比如 1px 2px, 5px 6px 是包含 1px 2px 与 5px 6px 两个值列表的值列表。如果内外两层值列表使用相同的分隔方式，要用圆括号包裹内层，所以也可以写成 (1px 2px) (5px 6px)。当值列表被编译为 CSS 时，Sass 不会添加任何圆括号，因为 CSS 不允许这样做。(1px 2px) (5px 6px)与 1px 2px 5px 6px 在编译后的 CSS 文件中是一样的，但是它们在 Sass 文件中却有不同的意义，前者是包含两个值列表的值列表，而后者是包含四个值的值列表。

声明默认变量

Sass 的默认变量仅需要在值后面加上 !default 即可。
Sass 的默认变量一般是用来设置默认值，然后根据需求来覆盖的，覆盖的方式也很简单，只需要在默认变量之前重新声明下变量即可。

混合宏

主要作用：重用 CSS 代码
使用"@mixin"来定义混合宏，它可以传递参数
调用混合宏，使用"@include mixin_name"
混合宏的缺点：容易造成代码的冗余，相同的代码会在编译出来的CSS文件出现多处

插值#{}

大括号里面可以是变量
插值的作用：
1、构建属性名字符串
2、构建选择器名称
但它不能用于构建变量名，另外在使用"@include"调用混合宏时，插值不能用于构建混合宏名称；使用"@extend"继承时，基类名称或者占位符名称可以用插值构建。
